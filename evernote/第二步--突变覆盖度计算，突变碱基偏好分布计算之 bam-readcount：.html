第二步--突变覆盖度计算，突变碱基偏好分布计算之 bam-readcount：

软件描述：
    bam-readcount是由MIT开发的用于统计bam文件中对应基因组位点上的碱基分布情况，该输出结果可用于二代测序数据的质量控制，相应代码目前托管于github上。

下载地址：
    https://github.com/genome/bam-readcount

使用方法：
    bam-readcount-f ref.fa some.bam -l example.bed -b 20 

常用参数说明：
    -q [ --min-mapping-quality ] ：参与计算的最小比对质量的reads，默认值0。
    -b [ --min-base-quality ] : 参与计算的最小碱基质量分数，默认值0
    -l [ --site-list ] : 参与统计的位点文件，bed文件格式
    -f [ --referene-fasta ] : 参考基因组fasta文件
    -d [ --max-count ] : 设置位点最大深度上限，避免内存不够，默认值为10000000
    -i [ --insertion-centric ] : 给出的indel中心化后的统计结果，位点上包含insertion的片段将不会计入统计。

输出文件列名详解：
    1. Chr: 染色体
    2. Position: 具体位置Reference_base:参考基因组碱基[ATCGatcg]，大写碱基表示参与蛋白质编码的位点，小写碱基表示非编码位点，其中Reference_base的大小写来自各自的reference fasta文件，大写表示该位点处于编码序列上，小写表示该位点处于内含子/UTR等的非翻译区。
    3. 突变碱基
    4. Depth：位点深度
    5. 接下来每列列代表每种不同碱基的信息,如A C G T N等，其中碱基的信息用“：”号分隔开，每个‘：’前对应的字段的意思见下。
        1)  Base:单个碱基或者插入缺失的碱基
        2)  Count:深度
        3)  avg_mapping_quality：平均mapping质量分数
        4)  avg_basequality：平均碱基质量分数
        5)  avg_se_mapping_quality：平均single ended的mapping质量分数
        6)  num_plus_strand:+/正义链上的reads数
        7)  num_minus_strand:-/负义链上的reads数
        8)  avg_pos_as_fraction:read上的平均位置分数（通过裁剪后的长度进行计算），这个读长的值被中心化的方法标准化（碱基都在序列中间的中心化后值为1，如果碱基都在序列尾部中心化后值为0）
        9)  avg_num_mismatches_as_fraction:每条reads上的每个base的平均错配率
        10) avg_sum_mismatch_qualities:每条reads上错配碱基质量分数和的平均数
        11) num_q2_containing_reads:时在3’末端是Q2 run的reads数
        12) avg_distance_to_q2_start_in_q2_reads:Q2 run的开始到位点的平均距离（没裁剪的读长作为分子）
        13) avg_clipped_length:裁剪后reads的平均长度
        14) avg_distance_to_effective_3p_end：到序列3’末端的的平均距离
