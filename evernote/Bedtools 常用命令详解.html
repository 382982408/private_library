1.Genome arithmetic /基因组算法

1.1 bedtools intersect

bedtools intersect【options】 -a <Afile: bed/gff/vcf> –b <Afile: bed/gff/vcf>
Options:
-abam:如果A输入的是Bam文件，则结果以Bam文件输出。
-ubam:以非Bam格式输出结果文件（如果没有设此参数，则默认Bam格式输出）。
-bed:当输入文件为Bam即使用了-abam参数时，改变输出格式为Bed文件格式。
-wa:即在结果文件中强行命令报告出其在原始文件A中的信息。
-wb:即在结果文件中强行命令报告出其在原始文件B中的信息。
-loj:如果A文件的某些区间在B文件中找不到overlap关系，则报告无效信息。
-u:仅报告A中与B有overlap的区间信息（只会报告A中原始信息）。
-c: 报告A中原始信息，无overlap以0表示，并报告overlap数量。
-v:仅报告A中与B没有overlap的区间信息。与-u参数相反。
-f:最小overlap（占总长度比例）参数设置，默认值为1E-9 。
-r:表示overlap在B文件中所占的比例必须与在A文件中所占的比例一致：如果overlap占A区间的90%以上，那么它也必须占B区间的90%以上。
-s：只进行同链比对。
-S：只进行异链比对。
实例：
cat A.bed
       chr1 100  200
       chr1 1000       2000
cat B.bed
       chr1 150 250  
bedtools intersect –a A.bed –b B.bed
       chr1 150  200
bedtools intersect –a A.bed –b B.bed –wa –wb
       chr1 100  200  chr 150  200

1.2 bedtools subtract

bedtools subtract [OPTIONS] -a <bed/gff/vcf> -b <bed/gff/vcf>
Options：
-f：最小overlap（占总长度比例）参数设置，默认值为1E-9 。
-s：只进行同链比对。
-S：只进行异链比对。
-A：将与B有overlap关系的A区间整个剔除。（原本默认是只剔除有overlap的区间）。
此功能与bedtools intersect 相反，取得是A中与B不相交的区间。

1.3 bedtools complement

bedtools complement  [OPTIONS] -i <bed/gff/vcf> -g <genome>
Geneme format：
chr1    249250621
chr2    243199373
用于寻找所有未被测序覆盖的基因组区间。
此功能与bedtools subtract的差异？
bedtools subtract的输出结果是Abed的子集。另外B文件是区间。
bedtools complement的输出结果是Bgenome的子集。而B文件是基因组格式，是大小。
示例：bedtools complement –I repeatMasker.bed –g hg19.genome

1.4 bedtools window

bedtools window 【options】-a <Afile: bed/gff/vcf> –b <Afile: bed/gff/vcf>
Options:
-abam:如果A输入的是Bam文件，则结果以Bam文件输出。
-ubam:以非Bam格式输出结果文件（如果没有设此参数，则默认Bam格式输出）。
-bed:当输入文件为Bam即使用了-abam参数时，改变输出格式为Bed文件格式。
-w: 将A区间两端延伸一定距离后，再与B文件进行overlap比对。默认值为1000.
-l: 将A区间上游延伸一定距离后，再与B文件进行overlap比对。默认值为1000.
-r: 将A区间下游延伸一定距离后，再与B文件进行overlap比对。默认值为1000.
       注意：-w不能与-l或者-r同时使用。
-sw:校正正负链信息后再进行距离延伸，例如当使用-l 500参数时，如果正链条则在上游延伸500bp；如果负链则在下游延伸500bp。
-sm: 只进行同链比对。
-Sm: 只进行异链比对。
-u:仅报告A中与B有overlap的区间信息。
-c:报告A中所有信息，与B没有overlap关系的区间以0表示。
-v:仅报告A中与B没有overlap的区间信息。与-u参数相反。
实例：bedtools window -a Window_interval.txt -b /sam2bed.txt -w 1 -c > bed_depth_stat.txt
bedtools intersect 与bedtools window用法及其相似，不同点在于：
  1. bedtools intersect能设置overlap参数，这是bedtools window做不到的。
  2. bedtools window能对区间进行延伸后比较，这是bedtools intersect做不到的。

1.5 bedtools slop 和 bedtools flank

bedtools slop [OPTIONS] -i <bed/gff/vcf> -g <genome> [-b <int> or (-l and -r)]
bedtools flank [OPTIONS] -i <bed/gff/vcf> -g <genome> [-b <int> or (-l and -r)]
Options：
-b：两端延伸：
-l：左端延伸。
-r：右端延伸：
-s：校正正负链信息后再进行距离延伸，例如当使用-l 500参数时，如果正链条则在上游延伸500bp；如果负链则在下游延伸500bp。
bedtools slop、bedtools flank、 bedtools window三者之间的区别？？？
Bedtools window只是延伸距离用于比对，即最终输出还是原区间信息。
Bedtools slop 延伸后区间会发生变化，即最终输出会是改变后的区间。
Bedtools flank 与slop不同的是，slop只是改变原区间，而flank延伸会产生新的区间。

1.7 bedtools sort

bedtools sort  [OPTIONS] -i <bed/gff/vcf> （值得一提的是sort的帮助文档需要-h）
Options:（默认以染色体排序后，按区间的起始位置升序排列）
-sizeA：根据片段大小升序排列。
-sizeD：根据片段大小降序排列。
-chrThenSizeA：先按染色体排序，后根据片段大小升序排列。
-chrThenSizeD：先按染色体排序，后根据片段大小降序排列。
-chrThenScoreA：先按染色体排序，后根据分值大小升序排列。
-chrThenScoreD：先按染色体排序，后根据分值大小降序排列。

1.8 bedtools closest

bedtools closest【options】-a <Afile: bed/gff/vcf> –b <Afile: bed/gff/vcf>
Options:
-s：只进行同链比对。
-S：只进行异链比对。
-d: 除报告最近区间信息，还会额外添加一列输出其与区间的距离信息，overlap距离为0.
-D：与-d相似但是如果最近区间在A区间的上游，则距离为负。
-io：忽视有overlap的区间。
-iu：与-D同时使用，忽略A区间上游信息。
-id：与-iu相似，忽略A区间下游信息。
-t：如果有B中有多个信息与A区间最为亲密。
       -all：默认全部输出
       - "first"：只输出第一条
       - "last"：只输出最后一条
-N：要求匹配区间reads信息不一样，如bed文件则比较第四列。
              

1.9 bedtools map

bedtools map [OPTIONS] -a <bed/gff/vcf> -b <bed/gff/vcf>
Options:
-c：指定B文件的列数与A区间进行比较，默认第五列。
-o：指定-c对应列数的参数属性。
              min；sum；max；mean；median等，默认指定sum
-f：最小overlap（占总长度比例）参数设置，默认值为1E-9 。
-r:表示overlap在B文件中所占的比例必须与在A文件中所占的比例一致：如果overlap占A区间的90%以上，那么它也必须占B区间的90%以上。
-s：只进行同链比对。
-S：只进行异链比对。
bedtools map -a last -b copy -c 4 -o mean
注：所有输出文件必须预先以染色体进行排序。

1.10 bedtools genomecov

bedtools genomecov  [OPTIONS] -i <bed/gff/vcf> -g <genome>
Options:
-ibam：指定输入文件为bam格式。
-d：计算并输出基因组每个位点的深度。
-dz：计算基因组每个位点的深度，但只输出非0的位点。
-bg：以bed格式报告深度。
-bga：同-bg，但是会输出所有0X的结果。
-strand：-+，--：分别计算正负链的结果。
-5：以5"位置代替整个区间进行计算。
-3：以3"位置代替整个区间进行计算。
-max：与-d等相反，会将所有深度大于等于max值得点进行合并计算。
-scale：常数因子，深度报告前会与常数因子相乘。
                            

1.11 bedtools merge

bedtools merge [OPTIONS] -i <bed/gff/vcf>
Options：
-s：只进行同链合并。
-n：报告合并文件的个数。
-d：设置最小距离。默认值为0，即合并所有overlap。
-nms：列出所有合并区间信息，并以分号分隔。

1.12 bedtools annotate

bedtools annotate  [OPTIONS] -i <bed/gff/vcf> -files FILE1 FILE2..FILEn
Options:
-names:给每列结果添加标识。
-counts：计算每个文件中与区间有overlap的个数。
-both：展示overlap数量与百分比信息。（默认只展示百分比）
-s：只进行同链比对。
-S：只进行异链比对。
示例：bedtools annotate -i B.bed -files A.bed C.bed -both -names AA CC
结果示例
# AA_cnt  AA_pct  CC_cnt  CC_pct

NC_000001.10    160     180     ##      1      1.000000       0     0.000000
NC_000001.10    170     200     ##      1      0.333333       0     0.000000

2．Multi-way file comparisons /文件间的不同比较方法

2.1 bedtools multiinter

bedtools multiinter  [OPTIONS] -i FILE1 FILE2 .. FILEn
Options：
-names：给每个文件添加标识。
-header：添加标题行。
-empty：以none报告没有交集的区间。
-examples：实例讲解。
详细用法请使用命令：
bedtools multiinter -examples

2.1 bedtools unionbedg

bedtools unionbedg  [OPTIONS] -i FILE1 FILE2 .. FILEn
Options：
-names：给每个文件添加标识。
-header：添加标题行。
-empty：以none报告没有交集的区间。
-examples：实例讲解。
详细用法请使用命令：
bedtools unionbedg-examples
此工具与bedtools multiinter非常相似，不同的是，bedtools multiinter比较的结果展示的是overlap个数。
              

3．Paired-end manipulation /pair-end reads格式转化

3.1 bedtools pairtobed

bedtools pairtobed  [OPTIONS] -a <bedpe> -b <bed/gff/vcf>
Options：
-abam:如果A输入的是Bam文件，则结果以Bam文件输出。
-ubam:以非Bam格式（Sam）输出结果文件（默认Bam格式输出）。
-bedpe：当输入文件是Bam文件时，以bedpe格式输出。
-f:最小overlap（占总长度比例）参数设置，默认值为1E-9 。
-s：只进行同链比对。
-S：只进行异链比对。
-type：
either（默认选项）：一个比对上或者两个都比对上。
neither：一对reads必须都比对不上。
both：一对reads必须都比对上，与neither相反。
xor：有且仅有一个read比对上。
notboth：只比对上一个或者都比对不上。
Ispan：第一个read的末尾和第二个read的起始之间比对上。
       Ospan：第一个read的起始和第二个read的末尾之间比对上。
Notispan：Ispan的补集。
Notospan：ospan的补集。

3.2 bedtools pairtopair

bedtools pairtopair  [OPTIONS] -a <BEDPE> -b <BEDPE>
-f:最小overlap（占总长度比例）参数设置，默认值为1E-9 。
-slop：对A区间进行延伸比对。
-ss：识别strand后slop延伸A区间时只往下游延伸。
-is：比对时忽略strand信息。
-rdn：忽略区间名进行比对。（默认时readname必须一样）
-type
       both(默认选项): 一对reads必须都比对上。
       neither：一对reads必须都比对不上。
either：一个比对上或者两个都比对上。
notboth：只比对上一个或者都比对不上。

5. Fasta manipulation/fasta文件处理

5.1 bedtools getfasta

bedtools getfasta  [OPTIONS] -fi <fasta> -bed <bed/gff/vcf> -fo <fasta>
options：
-fi：输入fa文件。
-bed：输入baed区间。
-fo：输出fa文件。(默认输出格式fasta)
-name：定义获得fasta的名字。
-tab：以tab键格式输出（chr1\tAAAACCC）。
-s：识别strand后进行匹配输出。
示例：
bedtools getfasta -fi /share/temp-cluster/Exome/hg19/hg19_reference_with_NC.fasta -bed 100K.all.bed -fo 100K.all.fa

5.2 bedtools maskfasta

bedtools maskfasta [OPTIONS] -fi <fasta> -fo <fasta> -bed <bed/gff/vcf>
Options：
-fi：输入fa文件。
-bed：输入baed区间。
-fo：输出fa文件。
-soft：将区间序列改为小写字母标红展示。
getfasta和maskfasta的区别：
getfasta是提取bed区间的碱基序列为新的fasta文件。
而maskfasta只是在fasta文件中标识（默认标红字体）出bed区间序列。
